# Antigravity 双色球深度演进系统 - 项目核心记忆 (PROJECT_MEMORY)

> **文档目的**: 记录本项目的演进深度、核心算法逻辑、因果假设与重写机制，以便在开启新对话时，Gemini 可以迅速对齐上下文环境并接管系统运作。

## 1. 确定性研究共识 (Research Consensus)

*   **根本目标**: 从双色球看似绝对随机的数字生成过程中，寻找并提取极其微弱的物理扰动和统计偏差，从而将“纯不可预知的盲猜”降维为“基于多维贝叶斯与对抗生成的高概率推演”。
*   **计算哲学**: 抛弃传统的经验主义筛号策略（如单一看尾号、跨度），采用 **千万级重火力算力覆盖 (Tensorized Monte Carlo)** 结合 **数据自主演进修正闭环**。
*   **评判标尺**: 所有的推荐结果必须经过历史数据回测，当系统的预测确定性（`Confidence Score`）发生衰退时，需要自主驱动代码级的变异。

## 2. 核心架构设计与演进深度

系统已不再是简单的爬虫与规则过滤脚本，而是演变为一套具备物理知觉与对抗神经的自动化守护引擎。

### 2.1 物理熵值算法逻辑 (Physical Entropy Logic)
*   **模块**: `core/crawler.py`
*   **逻辑机制**: 我们不再认为摇奖机是绝对封闭的理想数学系统。代码除了抓取历史出号记录，还会自动为每一期历史注入模拟的（或真实拉取的）**物理环境变量**，例如：
    *   **温度 (Temperature)**: 影响摇奖球的分子运动与材料弹性系数。
    *   **气压 (Pressure)**: 影响空气阻力与球体浮动轨迹。
*   **数据结构扩展**: 原本单一的 `[Red1..6, Blue1]` (7维) 特征，被扩展拼接成了更高维度的张量矩阵（例如 9维：包含 `entropy_temp` 与 `entropy_pressure`）。让模型学习“环境扰动”与“开奖结果”的隐含映射。

### 2.2 极速算力：张量化蒙特卡洛引擎 (Tensorized Monte Carlo)
*   **模块**: `core/inference.py` (封装于 `StrategyEngine`)
*   **逻辑机制**: 传统 Python 的 `for` 循环存在性能瓶颈，百万次运算需数十秒。
*   **演进深度**: 彻底改写为 Pure PyTorch Tensors 向量化计算。通过大规模矩阵相乘（`torch.matmul`）并行评估组合。
*   **性能基准**: 当前算力达到约 **158 万次组合/秒**。每次云端拟合下发权重后，系统毫不费力地在千万级以上的组合海报中筛选出打分（点积拟合度）最高的绝对 Top-1。

### 2.3 生成对抗网络模型 (GANs Architecture)
*   **模块**: `models/gan_adversary.py`
*   **逻辑机制**:
    *   **判别器 (Discriminator)**: 构建了基于多层感知机 (MLP) 的神经网络。
    *   `Real Data`: 来自于包含物理熵的历史真实开奖记录。
    *   `Fake Data`: 蒙特卡洛引擎生成的伪随机全覆盖序列。
*   **对抗目的**: 每经过一个周期，系统在后台执行 `train_step` 对抗训练。让判别器学会区分“看起来像真彩票的组合”与“绝对数学随机”，从而为最终产生的组合计算一个 **置信度评分 (Confidence Score)**。得分越接近 1.0，说明生成的号码组合的物理分布越贴近历史的真实轨迹。

### 2.4 因果分析与规则重写
*   **因果引擎**: `core/causal_reasoning.py`。负责统计反思。例如，当张量引擎暴力算出 5 连号的超级异常组合时，因果模块会将其判定为违背物理统计预期的 `[failed]`，形成逻辑刹车。
*   **回测监督**: `backtest.py`。当检测到当前过滤逻辑准确率太低（如 50%）时，强制重写 `strategies/default_filter.py`。
*   **大模型自动运维**: `core/llm_evolver.py`。利用 Google AI SDK (`google-genai`) 自动读取系统 `execution.log` 的报错或失分片段，由大模型自动推理并生成修复性 Python 代码直接覆盖物理文件，实现代码**自动变异与进化**。

## 3. 生产环境自动化运维
*   **模块**: `daemon_runner.py`
*   **当前状态**: 现已支持自主挂机运行 (`run_forever`)。默认每间隔 1 小时自动走通一遍：**数据采集 -> 云端上传 -> GANs 对抗更新 -> 权重下发 -> 万次推演 -> 波动校验** 的生息循环。
*   **报警策略**: 只有当本期 GAN 置信度评分较历史 5 期均值的波动幅度（`fluctuation`）超过 **20%** 时，才会以 `logger.warning` 在终端触发告警通知，达到免打扰的长期潜伏推演。
